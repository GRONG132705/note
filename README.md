## 使用建议

- 如果是面试复习：先看每个模块的总纲题，再看高频细题。
- 如果是系统学习：先看模块导航、知识地图和总结，再进入题目细节。
- 如果是项目复盘：优先看“典型业务场景”“学习提示”“追问答案”。

## 学习方法

- 第一遍先按文件顺序通读，只抓每份资料的模块导航、总目录和本模块总结，先建立全局框架。
- 第二遍按模块学习，每看完一个模块，就自己口头复述一遍“这组题到底在解决什么问题”。
- 第三遍开始按题目深挖，把“核心讲解”和“深入理解”连起来看，再顺手过“追问答案”。
- 第四遍进入输出阶段，尽量用自己的项目经历去套“典型业务场景”和“学习提示”，把知识点讲成链路。

## 分阶段路线

- 基础主线：`01 数据库 -> 02 缓存 -> 04 消息队列 -> 11 操作系统与计算机网络`
- 架构主线：`03 微服务 -> 05 分布式锁 -> 06 分布式事务 -> 09 高并发系统设计`
- Java 主线：`07 JVM -> 08 Java 并发 -> 12 Spring 生态`
- 面试表达主线：`10 项目表达与故障排查`

## 复习节奏建议

- `20 分钟`：只看每份资料的模块导航、总目录和每个模块的总纲题。
- `1 小时`：挑 1 到 2 份资料，按模块顺序读“核心讲解 -> 深入理解 -> 追问答案”。
- `半天`：按一条主线串联复习，比如“数据库 + 缓存 + MQ + 分布式事务”。
- `面试前`：优先看“学习提示”“典型业务场景”和你自己最容易卡壳的几道总纲题。
