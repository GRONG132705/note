# 凤凰架构

### 明确自身定位，选择性阅读

这本书的内容非常庞大，涵盖了从单体到微服务再到云原生的演进。作者在“前言”中明确指出了不同部分的受众，建议你根据自己目前的职位和技术阶段来侧重阅读 ：

- **初学者 / 正向微服务转型者：** 重点阅读**第一部分（演进中的架构）**，理解单体、SOA、微服务、无服务等概念的来龙去脉和优缺点 。
- **研发工程师 / 架构师：** 重点攻克**第二部分（架构师的视角）\**和\**第三部分（分布式的基石）**。这两部分是核心基本功，包含RPC、REST、分布式事务（XA、TCC、SAGA等）、服务发现、流量治理（熔断、限流）、分布式共识算法（Paxos、Raft、Gossip）等必备知识 。
- **运维工程师 / 平台开发人员：** 重点钻研**第四部分（不可变基础设施）**。这里涉及Docker、Kubernetes、持久化存储、调度以及Service Mesh（服务网格）的核心原理 。
- **技术决策者 / 研发主管：** 除了第一部分，务必精读**第五部分（技术方法论）**，理解康威定律、微服务拆分的边界以及如何治理系统复杂性 。

### 学习流程

| **学习阶段** | **核心领域**                     | **具体功能点（原子级拆解）**                                 | **解决的核心痛点**                             |
| ------------ | -------------------------------- | ------------------------------------------------------------ | ---------------------------------------------- |
| **第一站**   | **远程服务调用** (Communication) | 1. **序列化**：JSON、Protobuf、Hessian 对比 2. **I/O 模型**：BIO vs NIO (Netty 原理) 3. **连接管理**：长连接保持、连接池调优 | 解决“怎么说话”：跨进程通信的性能与协议兼容性。 |
| **第二站**   | **事务与一致性** (Consistency)   | 1. **补偿模式**：TCC (Try-Confirm-Cancel) 实现 2. **长事务**：Saga 状态机与反向补偿逻辑 3. **幂等性**：去重表、乐观锁、分布式 ID | 解决“账对不上”：保证多数据库操作的最终正确性。 |
| **第三站**   | **发现与网关** (Navigation)      | 1. **注册中心**：心跳续约、服务剔除、自我保护 2. **路由转发**：路径匹配、动态权重、断域处理 3. **统一入口**：跨域(CORS)、鉴权、请求聚合 | 解决“你在哪儿”：自动发现实例并提供统入口。     |
| **第四站**   | **流量治理** (Governance)        | 1. **限流算法**：令牌桶、漏桶、滑动窗口 2. **熔断状态**：Closed / Open / Half-Open 切换 3. **隔离策略**：线程池隔离、信号量隔离(舱壁模式) | 解决“系统过载”：防止局部故障引发系统雪崩。     |
| **第五站**   | **可观测性** (Observability)     | 1. **链路追踪**：TraceId 传递、采样率、拓扑图 2. **度量指标**：P99/P95 延迟、QPS、JVM 监控 3. **结构化日志**：ELK 检索、日志染色、Context 注入 | 解决“黑盒黑产”：在复杂链路中精准定位故障点。   |
| **第六站**   | **安全性** (Security)            | 1. **身份认证**：JWT 无状态校验、OAuth2 流程 2. **细粒度授权**：RBAC 模型、URL 级别权限 3. **传输安全**：mTLS 双向证书、敏感字段加密 | 解决“内网裸奔”：确保请求合法且不能越权操作。   |
| **第七站**   | **配置中心** (Configuration)     | 1. **动态刷新**：Hot-Reload (配置修改秒级生效) 2. **版本治理**：配置回滚、灰度发布配置、多环境隔离 3. **安全加固**：配置项加密存储 (如 DB 密码) | 解决“改配即启”：成百上千实例的配置统一管理。   |
| **第八站**   | **部署与编排** (Orchestration)   | 1. **容器化**：Docker 镜像制作、多阶段构建 2. **健康检查**：Liveness(存活) 与 Readiness(就绪) 3. **发布模式**：蓝绿发布、金丝雀(灰度)、滚动更新 | 解决“运维地狱”：实现环境一致性与自动化扩缩容。 |
